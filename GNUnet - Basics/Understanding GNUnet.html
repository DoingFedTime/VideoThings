Understanding GNUnet

A secure, distributed, and privacy-preserving network protocol stack

The Internet is Broken

The conventional Internet was not designed with security in mind. It's like a system of roads with potholes and highwaymen all over the place. Your data can be hijacked, damaged, or stolen without your knowledge.

GNUnet aims to replace the old insecure Internet protocol stack with a network protocol stack for building secure, distributed, and privacy-preserving applications.

Core Principles of GNUnet

Free Software

GNUnet is implemented as Free Software, giving users the freedom to run, study, share, and modify the software.

Privacy Protection

GNUnet minimizes the amount of personally identifiable information exposed during network operations.

Decentralization

The network is fully distributed and resilient to external attacks and rogue participants.

Self-Organization

GNUnet is self-organizing and doesn't depend on administrators or centralized infrastructure.

Transparency

Users are informed which other participants have to be trusted when establishing private communications.

Openness

GNUnet is open and permits new peers to join the network freely.

GNUnet: A Peer-to-Peer Network

GNUnet is a peer-to-peer framework for secure, decentralized networking. Watch how data flows through this encrypted network.

End-to-End Encryption

All communication is encrypted, protecting your data from interception.

Decentralized Routing

No central authority controls the network, making it resistant to censorship.

Metadata Protection

GNUnet hides who is communicating with whom, protecting your privacy.

GNUnet's LEGO Architecture

GNUnet's architecture is based on LEGOs - interchangeable components that can be stacked together to create complex systems. Each component is isolated from others for security and reliability.

Application Layer

User interfaces and daemons that interact with the user

Services Layer

Core services offering APIs and functionality

Transport Layer

Secure, encrypted communication between peers

Foundation Layer

Core libraries and utilities that all components rely upon

Hover over each layer to see its connections and responsibilities.

Key Subsystems

GNUnet consists of multiple subsystems that work together to create a secure and private network. Each subsystem handles specific aspects of the network's functionality.

Distributed Hash Table (R5N)

A secure way to store and retrieve data across the network without central servers.

CADET

End-to-end encrypted messaging layer similar to SCTP for secure communication.

GNU Name System (GNS)

Decentralized and secure alternative to DNS for naming resources.

Transport

Pluggable system for network communication supporting various protocols.

GNUnet vs. Traditional Internet

Traditional Internet

  • Centralized infrastructure vulnerable to censorship and control
  • Metadata exposure revealing who communicates with whom
  • Insecure routing allowing traffic interception
  • Dependence on trusted third parties

GNUnet

  • Decentralized peer-to-peer network resistant to censorship
  • Metadata protection through encryption and routing techniques
  • Secure end-to-end encrypted communication
  • Self-organizing without need for trusted third parties

Benefits of GNUnet

Privacy Protection

GNUnet protects both content and metadata, ensuring that your communications remain private from end to end.

Censorship Resistance

The decentralized design makes it extremely difficult for anyone to censor or shut down the network.

Freedom and Control

As free software, GNUnet puts you in control of your data and communications, respecting your essential freedoms.
